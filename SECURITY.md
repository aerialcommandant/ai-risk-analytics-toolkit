# Security Policy

## Supported Versions

This project is currently in early development. Security improvements and vulnerability reports are welcome.

## Reporting a Vulnerability

If you find a vulnerability, please open a GitHub issue with a clear description of the problem.

Please do not include secrets, passwords, private keys, or sensitive personal data in public issues.

## Security Focus Areas

This project aims to improve:

- Safe handling of data files
- Secure use of API keys and environment variables
- Dependency review
- Risk scoring transparency
- Secure maintainer workflows
